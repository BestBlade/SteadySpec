# Governed Proposal Path

Use this path when a normal proposal would be too loose because direction, boundary, validation, or architecture can drift.

1. Run or summarize `steadyspec-explore` until the problem is named.
2. Run `steadyspec-context-archaeology` if old structure or project history may affect the proposal.
   Do not turn inferred history into a constraint.
3. Run `steadyspec-grill` if the plan has unresolved decision dependencies.
4. Run `steadyspec-debate` for direction.
5. Run a second boundary debate focused on scope, touched layers, evidence, stop conditions, and non-goals.
6. Write findings before proposal artifacts:
   - direction findings
   - historical constraints and context gaps
   - grilled decision dependencies
   - boundary findings
   - accepted risks
   - parked long-tail concerns
   - unresolved items
   - evidence requirements
7. Generate proposal/design/spec/tasks or equivalent substrate artifacts from findings.
   Do not let artifacts exceed findings or erase rejected alternatives future agents may repeat.
8. Verify every high-impact decision traces to findings.

Report findings, artifact locations, key boundary, stop conditions, unresolved risks, and readiness for `steadyspec-apply`.
